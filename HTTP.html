HTTP and HTML

About

HTTP

Hypertext Transfer Protocol (HTTP) is an application-layer protocol for transmitting hypermedia documents, such as HTML. It was designed for communication between web browsers and web servers, but it can also be used for other purposes. HTTP follows a classical client-server model, with a client opening a connection to make a request, then waiting until it receives a response.

HTML

HTML (HyperText Markup Language) is the most basic building block of the Web. It defines the meaning and structure of web content. Other technologies besides HTML are generally used to describe a web page's appearance/presentation (CSS) or functionality/behavior (JavaScript).


Components of HTTP-based systems

HTTP is a client-server protocol, which means that just one entity, the user-agent, sends requests (or a proxy on behalf of it). The user-agent is often a Web browser, but it may be anything, such as a robot that crawls the Internet to populate and maintain a search engine index. Each individual request is submitted to a server, which processes it and responds with a response. Between the client and the server, there are a number of entities known as proxies that fulfill various functions, such as acting as gateways or caches.

Between a browser and the server that handles the request, there are other computers: routers, modems, and so on. These are concealed in the network and transport layers thanks to the Web's tiered architecture. At the application layer, HTTP is on top. The underlying layers are essentially unimportant to the explanation of HTTP, despite their importance in detecting network problems.



HTML

HTML elements are the components that make up HTML pages. Images and other objects, such as interactive forms, can be embedded in the generated page using HTML techniques. HTML allows you to construct organized documents by indicating structural semantics for text elements like headers, paragraphs, lists, links, quotations, and other elements. Tags, which are written in angle brackets, separate HTML components. Tags like image and insert material into the page immediately. Other tags, such as break line, surround and give information about document content, and may comprise sub-elements such as other tags. The HTML tags are not shown by browsers, but they are used to comprehend the page's content.
